## 해수면 상승 경로 시뮬레이션 프로젝트 2인 협업 로드맵

### Phase 1. 설계 및 기초 데이터 확보 (1~2주)

- **사용자 (Data Analyst):**
    - **공간 데이터 수집:** 대상 지역(서울 등)의 DEM(고도 데이터) 및 도로망 데이터(OSM, SHP) 확보.
    - **시나리오 정의:** 해수면 상승 범위(예: 0.5m ~ 5.0m) 및 침수 판단 기준 수립.
- **친구 (Backend Dev):**
    - **시스템 아키텍처 설계:** FastAPI 프레임워크 및 PostGIS DB 스키마 설계.
    - **지도 라이브러리 선정:** Mapbox 또는 Leaflet 기반의 인터페이스 프로토타입 검토.

---

### Phase 2. 핵심 로직 및 DB 구축 (2~3주)

- **사용자 (Data Analyst):**
    - **침수 분석 엔진:** 고도 데이터를 활용해 해수면 높이별 도로 폐쇄 여부 판단 알고리즘 구현(Python).
    - **경로 탐색 알고리즘:** NetworkX 기반의 가중치 그래프 생성 및 최단 경로 검색 함수 개발.
        
- **친구 (Backend Dev):**
    - **DB 마이그레이션:** 전처리된 지리 데이터를 PostGIS에 적재 및 공간 인덱싱 최적화.
    - **기본 API 구현:** 출발/도착 좌표 및 해수면 파라미터를 수신하는 REST API 엔드포인트 개발

---

### Phase 3. 인터렉티브 통합 및 프론트엔드 (2주)

- **사용자 (Data Analyst):**
    - **결과 데이터 정제:** 시뮬레이션 결과를 GeoJSON 형식으로 변환하여 백엔드 전달 최적화.
    - **로직 고도화:** 단순 거리 외에 '침수 확률' 등에 따른 페널티 부여 로직 튜닝.
        
- **친구 (Backend Dev):**
    - **프론트엔드 개발:** 지도 UI상에서 사용자가 슬라이더로 해수면 높이를 조절하는 기능 구현.
    - **경로 시각화:** 분석 결과(침수 구역, 우회 경로)를 지도 위에 오버레이 렌더링.

---

### Phase 4. 검증 및 배포 (1주)

- **사용자 (Data Analyst):**
    - **모델 검증:** 실제 저지대 지형과 시뮬레이션 침수 구역의 일치 여부 샘플 테스트.
    - **데이터 요약:** 경로 변경에 따른 거리 증가량 등 통계 지표 산출.
        
- **친구 (Backend Dev):**
    - **배포:** 클라우드 서버(GCP, AWS 등) 또는 정적 호스팅 서비스에 배포.
    - **성능 최적화:** 지도 데이터 로딩 속도 개선 및 예외 처리(에러 핸들링).
        

---

### 5. 비판적 검토 및 병목 지점
- **데이터 용량 문제:** 고해상도 DEM 데이터는 메모리 점유율이 높음. 분석 단위를 타일(Tile)화하거나 사전 연산(Pre-calculated)된 데이터셋 활용 권장함.
- **의존성 관리:** 사용자의 Python 로직과 친구의 API 서버 간 입출력 규격(Interface)을 초기 단계에서 확정해야 통합 비용을 줄일 수 있음.
- **비영리성 제약:** Mapbox 등 상용 맵 라이브러리 사용 시 무료 티어 한도를 반드시 확인해야 함. (OpenStreetMap 기반 권장)
